# 医疗器械 ERP 系统开发环境迁移与无缝衔接指南

本指南旨在帮助您在更换账户后，能够快速搭建环境并继续进行开发工作。

## ⚠️ 核心开发准则 (必须严格遵守)

1. **UI 布局保护原则**：当前系统的 UI 页面和布局已定稿，用户非常满意。**严禁在未收到明确指令的情况下进行任何 UI 或布局的修改**。
2. **高效执行原则**：所有指令必须减少不必要的步骤，追求最快完成，严禁冗余操作，**最大限度节省积分消耗**。
3. **真实数据驱动原则**：所有页面开发的第一原则是**建立真实数据库数据**。严禁仅使用前端 Mock 数据，必须通过真实数据的增删改查来测试系统逻辑并排查漏洞。

## 4. 操作日志 (Audit Logs)

系统已具备初步的操作日志记录机制，用于追踪关键数据的变更。

- **后端存储**: 数据库 `operation_logs` 表。
- **前端记录**: 使用 `useOperationLog` 钩子。
- **已覆盖模块**: 
  - **产品管理**: 新增、编辑、删除操作均会记录。
  - **用户管理**: 新增、编辑、删除操作均会记录。
- **查看位置**: 系统设置 -> 操作日志 (需确保该页面已对接 `trpc.logs.list` API)。
